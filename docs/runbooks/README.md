# Runbook Summaries

This section contains public-safe runbook summaries for the Skod V2
transformation lab. They describe the operating model without publishing
private procedures, secrets, customer data, raw prompts, or low-level
logs.

## Public Runbooks

| Runbook | Focus |
|---|---|
| [AI team productivity loop](ai-team-productivity-loop.md) | How AI-assisted work is scoped, reviewed, measured, and improved. |
| [Observability](observability.md) | How operational visibility supports delivery governance. |
| [Local workers](local-workers.md) | How background work is treated as an operational concern. |
| [EmailConnect inbound](emailconnect-inbound.md) | How inbound email migration is governed safely. |

## Why This Matters

Runbooks turn good intentions into repeatable operating practice. In an
AI-assisted modernization program, they also reduce context drift: the
same decisions can be applied across tickets, reviews, and contributors.

## Not Public

Detailed commands, environment values, credentials, provider settings,
customer examples, raw prompt logs, and private incident details are not
published in this repository.
