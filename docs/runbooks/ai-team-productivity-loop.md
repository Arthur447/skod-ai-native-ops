# AI Team Productivity Loop

## Pattern

AI-assisted work is useful only when it improves delivery outcomes. The
loop starts from a scoped issue, uses a bounded AI task, lands in a
small PR, passes CI, receives human review, and records what was learned.

The metric is not prompt volume. The metric is better delivery: shorter
cycle time, less rework, better first-pass review quality, fewer escaped
defects, and more reusable knowledge.

## Why It Matters

This is AI-native governance because AI is integrated into the operating
model while authority stays explicit:

- issues define intent;
- CI catches mechanical defects;
- humans own architecture and acceptance;
- learning is converted into tests, runbooks, ADRs, or better scoping.

AI augments execution under human governance. It does not replace
engineering accountability.

## What Is Not Public

Raw prompts, low-level Codex logs, private ticket details, customer
data, cost traces, and sensitive implementation examples are not
published here.
